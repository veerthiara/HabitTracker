# HabitTracker Documentation

This site turns the repository docs into a readable handbook.

## Reading order

Start with the roadmap if you want the product and execution plan from the top:

1. Read [Project Plan](./roadmap/project-plan.md)
2. Continue through the phase roadmap documents
3. Use the implementation section to inspect what changed revision by revision
4. Use the architecture section when you need the system model quickly

## Sections

- **Roadmap**: the long-form plan and phase breakdowns
- **Implementation**: revision history for completed work
- **Architecture**: system overview and data model
- **Copilot**: prompts and repo guidance

## Source of truth

All long-form content lives in the repository under [`docs/`](https://github.com/veerthiara/HabitTracker/tree/main/docs). The site is only a reading layer on top of those Markdown files.
